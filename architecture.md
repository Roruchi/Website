# Architecture

This document describes the technical architecture and design decisions of the personal website.

---

## Overview

The site is a **static website** generated by [Eleventy (11ty)](https://www.11ty.dev/) and styled with [Tailwind CSS](https://tailwindcss.com/). It is deployed to **GitHub Pages** via a GitHub Actions CI/CD pipeline. There is no server-side runtime or database; all content is stored as Markdown files and compiled to static HTML at build time.

The repository also contains a **manual crosspost utility** for blog posts. That flow runs as a local CLI script, reads post front matter, generates LinkedIn copy, and can create Medium drafts through Playwright when a post is tagged for Medium distribution.

The repository also includes a **lightweight content and speaking workflow**. Articles remain in Eleventy's source tree. Private-by-default talks, opportunities, and submissions live in top-level `content/` directories outside the public input. A submission is the link between one talk and one opportunity. Small scripts scaffold content, validate relationships and metadata, generate a local dashboard, and scan built output for private values.

## Build Pipeline

The build runs in three sequential steps via `npm run build`:

```
npm run build:css   →   tailwindcss -i src/assets/input.css -o src/assets/style.css --minify
                            ↓
eleventy            →   reads src/, compiles Nunjucks + Markdown → _site/
                            ↓
npx pagefind        →   indexes _site/ → generates _site/pagefind/ (search index)
```

### CSS Build
Tailwind scans all `.njk`, `.md`, `.html`, and `.js` files under `src/` for class usage, then generates an optimized, minified CSS file at `src/assets/style.css`. This file is passed through to `_site/assets/style.css` via Eleventy's passthrough copy.

### Eleventy Build
Eleventy reads all source files and applies the following:
- **Templates** (`.njk`): compiled using Nunjucks with global data from `src/_data/`
- **Markdown** (`.md`): converted to HTML via markdown-it, front matter parsed as page data
- **Plugins**: syntax highlighting, RSS, reading time, sitemap, table of contents (see below)
- **Collections**: `post` and `training`, filtered to exclude draft content
- **Public talk data**: `src/_data/publicTalks.js` loads only talks with `visibility: public` and `showcase: true`, then removes private fields
- **Filters**: `readableDate`, `htmlDateString`, `head`, `relatedPosts`
- **Image shortcode**: `{% image %}` — generates responsive AVIF/WebP/JPEG at multiple widths via `@11ty/eleventy-img`

### Pagefind Index
After the Eleventy build, Pagefind crawls the `_site/` directory and produces a static search index at `_site/pagefind/`. The search page loads the Pagefind UI from this index at runtime.

### Crosspost Automation
Crossposting is intentionally kept outside the visitor-facing build and outside the GitHub Pages pipeline. `scripts/crosspost/publish.js` is meant to be run manually after a post is live and:
- Parses blog post front matter from `src/blog/*.md`
- Skips drafts and posts that already have an `originalUrl`
- Requires the `crosspost-medium` tag (configurable through `CROSSPOST_MEDIUM_TAG`) before attempting Medium
- Creates Medium drafts via Playwright using saved auth state
- Leaves the final Medium publish action to the site owner

### Content Workflow Automation
The authoring workflow is intentionally small and local-first:
- `npm run new:article` scaffolds a new `src/blog/*.md` file using either CLI flags or interactive prompts
- `npm run new:talk` scaffolds a private `content/talks/*.md` file
- `npm run new:opportunity` scaffolds a private `content/opportunities/*.md` file
- `npm run new:submission` links an existing talk and opportunity in `content/submissions/*.md`
- `npm run speaker:discover` previews configured public JSON/RSS/Atom sources; `-- --write` creates new opportunity files without overwriting existing content
- `npm run speaker:calendar` generates a local `.ics` file from opportunity deadlines and event dates
- `src/_data/content-taxonomy.js` defines the allowed pillars and tags
- `npm run check:content` validates metadata, relationships, internal links, image text coverage, public fields, and the no-em-dash rule
- `npm run dashboard:content` generates `.content-dashboard/index.html`, outside `_site`
- `npm run serve` sets `LOCAL_DASHBOARD=true`, serves `/dashboard/`, and shows its navigation link locally
- Production builds leave `LOCAL_DASHBOARD` unset, so private dashboard data is not loaded and the route has `permalink: false`
- `content/discovery-sources.json` is the explicit allowlist and mapping registry for public discovery sources
- `.speaker-calendar/` contains local calendar exports and is excluded from version control and deployment
- `npm run publish:check` runs validation, a full site build, and the built-output privacy scan
- `npm run qa:publish` aliases the publishing gate used in CI

## Deployment Pipeline

Deployments are triggered by every push to the `main` branch via `.github/workflows/deploy.yml`:

```
Push to main
    ↓
GitHub Actions: build job
    ├── Checkout code
    ├── Setup Node.js 20
    ├── npm ci
    ├── npm run build
    └── Upload _site/ as Pages artifact
    ↓
GitHub Actions: deploy job
    └── Deploy artifact to GitHub Pages
```

The live site URL is the GitHub Pages URL for the repository. The Eleventy `pathPrefix` is automatically derived from the `GITHUB_REPOSITORY` environment variable in `.eleventy.js` so that all asset and page paths work correctly under the repository sub-path.

The crosspost utility is not part of the static site runtime or deployment workflow. It depends on local environment variables for LinkedIn generation/publishing configuration and on Playwright storage state for Medium authentication.

Separate from deployment, `.github/workflows/quality.yml` runs on pull requests and pushes to `main` and executes `npm run qa:publish` so content changes are validated before and after merge with the same command used locally.

## Key Eleventy Configuration (`.eleventy.js`)

| Concern | Detail |
|---|---|
| **Plugins** | syntaxHighlight, TOC (h2/h3), RSS, readingTime, sitemap |
| **Image shortcode** | `{% image src, alt, sizes %}` — async, multi-format, lazy-loaded |
| **Collections** | `post`, `training` — filtered by `status: draft` or legacy `draft: true` |
| **`relatedPosts` filter** | Scores posts by shared tags; returns top N matching posts |
| **Speaking privacy** | Only public showcased talks enter sanitized global data; opportunities and submissions never enter Eleventy |
| **pathPrefix** | Derived from `GITHUB_REPOSITORY` env var; defaults to `/` locally |
| **Input/Output dirs** | `src/` → `_site/` |

## Design System

The visual design follows a **"Zen Garden / Washi Paper"** aesthetic:

| Token | Value | Usage |
|---|---|---|
| Background | `#f8f6f1` (`bg-washi`) | Page background |
| Primary text | Tailwind `stone-900` | Headings, body |
| Secondary text | Tailwind `stone-500`–`stone-600` | Captions, metadata |
| Accent / CTA | Tailwind `amber-500`–`amber-700` | Buttons, links, tags |
| Card backgrounds | `white` | Content cards |
| Font (UI) | Inter | Navigation, labels, UI elements |
| Font (prose) | Source Serif 4 | Blog post body text |
| Font (code) | JetBrains Mono | Code blocks, inline code |

There is intentionally **no dark mode** on this site.

## Client-Side Features

| Feature | Implementation |
|---|---|
| **Koi pond animation** | Canvas-based interactive animation; `src/assets/js/koi-pond.js` loaded on homepage |
| **Reading progress bar** | Inline `<script>` in `base.njk`; updates on `scroll` |
| **Mermaid diagrams** | Loaded from CDN (`mermaid@11`); replaces `<pre><code class="language-mermaid">` blocks with rendered SVG |
| **Mobile menu** | Inline `<script>` in `base.njk`; toggles `hidden` class |
| **Full-text search** | Pagefind UI loaded from `/pagefind/pagefind-ui.js` on the `/search/` page |
| **Syntax highlighting** | Server-side via `@11ty/eleventy-plugin-syntaxhighlight` (Prism); `prism-tomorrow.css` theme |

## Content Types

| Content Type | Source Directory | Eleventy Collection | Key Front Matter Fields |
|---|---|---|---|
| Blog posts | `src/blog/` | `post` | `title`, `slug`, `date`, `description`, `status`, `pillar`, `tags`, `relatedTalks` |
| Conference talks | `content/talks/` | Sanitized global data | `title`, `slug`, `status`, `visibility`, `showcase`, `audience`, `takeaways`, `relatedArticles` |
| Speaking opportunities | `content/opportunities/` | None | `name`, `slug`, `platform`, `format`, `deadline`, `status`, `topics`, `notes` |
| Submissions | `content/submissions/` | None | `talk`, `opportunity`, `status`, `submittedAt`, `decisionDate`, `abstractVersion`, `notes`, `public` |
| Training workshops | `src/trainings/` | `training` | `title`, `description`, `tags`, `duration`, `audience` |
| Static pages | `src/about/`, etc. | (none) | `title`, `description`, `layout` |
